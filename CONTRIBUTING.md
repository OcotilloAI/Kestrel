## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Create a new branch** for your feature or bugfix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make your changes**. Please ensure your code follows the existing style and conventions.
4.  **Test your changes**. Run any existing tests and add new ones if applicable.
5.  **Commit your changes**. Write clear and concise commit messages.
    ```bash
    git commit -m "Add a new feature to handle X"
    ```
6.  **Push to your fork**.
    ```bash
    git push origin feature/my-new-feature
    ```
7.  **Submit a Pull Request** (PR) against the `main` branch of the original repository.

## Pull Request Guidelines

*   Provide a clear description of the changes and the problem they solve.
*   Reference any related issues (e.g., "Fixes #123").
*   Ensure the code compiles and runs successfully.
*   All contributions are vetted by Ocotillo AI, Inc. prior to merging.

## Code of Conduct

Please be respectful and considerate in all interactions. We strive to maintain a welcoming and inclusive community.
